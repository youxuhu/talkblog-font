# TalkBlog 前端项目实验讨论报告

## 1. 实验目标

构建一个基于 Vue 3 + Vite 的现代化博客社区前端，核心功能包括人脸认证、博客管理、评论社交、后台审核及实时通知。

## 2. 实验结果分析

### 2.1 功能完成度

| 模块 | 完成情况 | 关键文件 |
|------|---------|---------|
| 人脸/密码认证 | 完整实现 | `HomeView.vue` — 摄像头采集、三种登录模式 |
| 博客 CRUD | 完整实现 | `BlogListView/DetailView/EditorView.vue` |
| 评论互动 | 完整实现 | `CommentItem.vue` — 回复/点赞/编辑/举报/图片/emoji/@提及 |
| 后台管理 | 完整实现 | `AdminCommentsView/UsersView/StatsView/ReportsView.vue` |

| 主题切换 | 完整实现 | `styles/theme.css` — CSS 变量暗色模式 |

### 2.2 技术选型讨论

**Vue 3 + Composition API**: 逻辑复用优于 Options API，`EmojiPicker` 和 `MentionList` 等小组件可独立维护。

**Pinia vs Vuex**: Pinia TS 支持和模块化更优，本项目 7 个 store 职责分明，但 `comment store` 混合了博客评论/管理评论/举报数据，职责过重。

**Element Plus + Pixel UI 双框架**: 共存增加了约 30% 的样式覆盖代码（见 `theme.css` 中同时适配两套组件的 `.dark` 规则），导致包体积增大。推测为 UI 迁移过渡产物。

### 2.3 数据流讨论

```
View → Store → Service (fetch) → Backend
```

服务层（`services/` 下 8 个文件）存在大量重复代码：`buildUrl()`、`requestJson()`、错误处理逻辑几乎完全相同。**违反了 DRY 原则**，应抽取公共 HTTP 模块。

### 2.4 评论系统设计讨论

**写后读模式**: 每次评论操作（创建/回复/编辑/点赞/删除）后 `emit('refresh')` 全量重载评论列表。优点是状态一致性好，缺点是：
- 网络开销大（每次重载整个列表）
- 用户体验差（操作后需等待加载）

**优化建议**: 改为**乐观更新** + 增量追加新评论。

### 2.5 认证安全性讨论

- JWT 存 `sessionStorage`：关闭标签页自动过期，优于 `localStorage`
- 人脸 base64 传输：需配合 HTTPS 防中间人
- 路由守卫三层校验：`requiresAuth` → `requiresAdmin` → 已登录自动跳转，覆盖面完整
- 评论区 `v-html` 渲染 `highlightMentions()`：存在 XSS 风险（`@<script>alert(1)</script>` 会被直接注入）

## 3. 结论

TalkBlog 前端功能完备，覆盖博客社区的全链路需求。主要改进方向：(1) 抽取公共 HTTP 模块消除代码重复；(2) 评论系统改为乐观更新；(3) 统一 UI 框架；(4) 增加 XSS 防护。
